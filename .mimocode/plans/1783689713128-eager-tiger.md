# VitePress 技术博客「栈里拾光」设计方案

## 现状

- **仓库**: `https://github.com/zhangqibo920/my-blog.git` (main 分支)
- **现有 CI**: `.github/workflows/deploy.yml` 基于 Hexo + `actions/deploy-pages`（需替换）
- **目录**: 仅 `.git/`、`.github/`、`.mimocode/`，无源码

## 架构

```
GitHub (源码)  ──push main──▶  GitHub Actions
                                    │
                              npm install → vitepress build
                                    │
                              wrangler pages deploy
                                    │
                              Cloudflare Pages
                                    │
                              *.pages.dev (免费域名)
```

**部署流程**: push 到 main → GitHub Actions 触发 → 安装依赖 + 构建 VitePress → Wrangler CLI 上传 dist 到 Cloudflare Pages。需要在 GitHub repo Secrets 中配置 `CLOUDFLARE_API_TOKEN` 和 `CLOUDFLARE_ACCOUNT_ID`。

## 实施步骤

### Step 1: 初始化项目
- 创建 `package.json`
  - `name`: `blog`
  - `scripts`: `docs:dev`, `docs:build`, `docs:preview`
  - `devDependencies`: `vitepress` (latest)
- Node >= 18

### Step 2: VitePress 配置 (`docs/.vitepress/config.ts`)
- **title**: `栈里拾光`
- **description**: 个人技术博客
- **lang**: `zh-CN`
- **nav**: 首页 `/`、指南 `/guide/`、博客 `/blog/`
- **sidebar**: guide 分组
- **socialLinks**: GitHub (`zhangqibo920`)
- **search**: 本地搜索 (local)
- **footer**: 版权信息

### Step 3: 首页 (`docs/index.md`)
- Hero: 标题「栈里拾光」+ 副标题 + 两个 CTA（开始阅读 / GitHub）
- Features: 3 个特性卡片（技术笔记 / 学习记录 / 开源分享）

### Step 4: 示例内容
- `docs/guide/getting-started.md`: VitePress 快速上手示例
- `docs/blog/index.md`: 博客文章列表占位页

### Step 5: 主题微调 (`docs/.vitepress/theme/`)
- `index.ts`: 继承默认主题 + 导入自定义样式
- `style.css`: 基础 CSS 变量覆盖（主色调等），保持简洁

### Step 6: 替换部署 workflow (`.github/workflows/deploy.yml`)
```yaml
name: Deploy to Cloudflare Pages
on:
  push:
    branches: [main]
concurrency:
  group: deploy
  cancel-in-progress: true

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm
      - run: npm install
      - run: npm run docs:build
      - uses: cloudflare/wrangler-action@v3
        with:
          apiToken: ${{ secrets.CLOUDFLARE_API_TOKEN }}
          accountId: ${{ secrets.CLOUDFLARE_ACCOUNT_ID }}
          command: pages deploy docs/.vitepress/dist --project-name=my-blog
```

### Step 7: `.gitignore`
排除: `node_modules/`、`docs/.vitepress/dist/`、`docs/.vitepress/cache/`

## Cloudflare 配置前置条件（用户需手动操作）

1. 登录 [Cloudflare Dashboard](https://dash.cloudflare.com)
2. 进入 Workers & Pages → Create → Pages → Connect to Git
3. 选择 GitHub 仓库 `zhangqibo920/my-blog`
4. 构建设置:
   - **Build command**: `npm run docs:build`
   - **Build output directory**: `docs/.vitepress/dist`
5. 获取 API Token: My Profile → API Tokens → Create Token → Cloudflare Pages 部署权限
6. 在 GitHub repo Settings → Secrets 中添加:
   - `CLOUDFLARE_API_TOKEN`: 上一步获取的 token
   - `CLOUDFLARE_ACCOUNT_ID`: Cloudflare Dashboard 右侧栏可见

## 验证

1. `npm install && npm run docs:dev` → 浏览器 `http://localhost:5173` 确认页面正常
2. `npm run docs:build` → 确认 `docs/.vitepress/dist/` 生成静态文件
3. 推送到 GitHub → Actions 运行成功 → Cloudflare Pages 部署完成
4. 访问 `https://my-blog.pages.dev` 确认线上可访问
